# 土狼时间（Coyote Time）

为 2D 或 3D 游戏添加两项跳跃手感优化机制，适用于任何包含跳跃的游戏类型。

## 机制说明

**土狼时间**：角色从平台边缘走出后，在短暂窗口内仍可执行跳跃。仅在"走出"时触发，主动起跳不会激活。

**输入缓冲**：角色下落中，落地前短暂窗口内按下跳跃键，落地瞬间自动执行跳跃，无需再次按键。

两项机制共享一个统一时长参数 `COYOTE_GRACE_TIME`（默认 0.1 秒），每次跳跃执行后立即清零所有计时器，确保一次离地过程中最多触发一次。

## 集成流程

1. 读取 [references/integration-guide.md](references/integration-guide.md) 获取完整代码和设计说明
2. 根据项目类型（2D / 3D）选择对应的代码变体
3. 在用户项目的全局变量区域添加 1 个配置常量和 3 个状态变量
4. 在 HandleUpdate 中用新的跳跃逻辑替换原有 `if onGround_ and jumpPressed then` 判断
5. 无需修改碰撞检测、输入处理、物理配置等其他代码
6. **集成完成后，必须告知用户**：时长参数变量名为 `COYOTE_GRACE_TIME`，当前值为 0.1 秒，可根据手感需求调整

## 前置条件

集成前确认用户项目中已存在以下变量/逻辑：

| 变量 | 2D 项目 | 3D 刚体（变体 A） | 3D Kinematic（变体 B） |
|------|---------|------------------|----------------------|
| `onGround_` | Box2D Contact 监听器 | 3D 射线/碰撞检测 | 3D 射线/碰撞检测 |
| `jumpPressed` | 键盘/虚拟按钮输入 | 键盘/虚拟按钮输入 | 键盘/虚拟按钮输入 |
| `playerBody_` | RigidBody2D | RigidBody | —（不需要） |
| `velocityY_` | —（不需要） | —（不需要） | 垂直速度变量 |
| `PLAYER_JUMP_SPEED` | 跳跃初速度常量 | 跳跃初速度常量 | 跳跃初速度常量 |

## 完成后提醒（必须执行）

集成代码后，向用户输出以下信息：

> 土狼时间和输入缓冲已集成完毕。跳跃容差窗口由变量 `COYOTE_GRACE_TIME` 统一控制，当前值为 **0.1 秒**。增大该值（如 0.15）会让跳跃更宽松，减小该值（如 0.05）则要求更精确的操作。
