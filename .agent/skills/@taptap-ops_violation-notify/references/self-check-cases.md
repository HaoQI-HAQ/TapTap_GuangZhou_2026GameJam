# 开发者自查案例库

> 本文件列出各违规类型的经典触发场景，帮助开发者在提审前主动排查。
> 所有案例均来源于官方违规定义，禁止编造文档中不存在的违规场景。

---

## 使用方式

按以下清单逐项对照自己的游戏，检查是否存在对应情况。每个案例标注了：
- **场景描述**：具体的违规行为表现
- **对应违规类型**：匹配的官方违规子类型编号
- **自查要点**：开发者需要检查的具体位置或内容

---

## 一、计费违规自查

### 案例 1.1：无版号游戏含付费功能

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏没有版号，但游戏内存在充值入口、内购商城、会员订阅、打赏/赞助功能等任何形式的付费内容 |
| 对应违规 | 1.1 偷开计费 |
| 自查要点 | ① 检查游戏是否持有版号 ② 若无版号，排查游戏内是否有任何付费入口（包括隐藏的、未上线的付费模块） ③ 注意"赞助"、"打赏"、"支持开发者"等变体也属于计费 |

### 案例 1.2：曾因计费违规被处理后再次出现

| 项目 | 内容 |
|------|------|
| 场景描述 | 已因偷开计费被处理过，整改后又重新加入了付费功能 |
| 对应违规 | 1.2 多次偷开计费 |
| 自查要点 | ① 确认是否有过偷开计费的处理记录 ② 若有，确保整改后不再包含任何付费内容 |

## 二、内容违规自查

### 案例 2.1：游戏含色情或性暗示内容

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内或商店页素材中含有裸露身体、性行为描写、性暗示画面或文字 |
| 对应违规 | 2.1 色情内容 |
| 自查要点 | ① 排查角色立绘是否有裸露/性暗示 ② 排查剧情文本是否有性行为描写 ③ 排查商店页截图和宣传图 |

### 案例 2.2：游戏含写实血腥画面

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内出现写实风格的鲜红血液、断肢、内脏等血腥暴力画面 |
| 对应违规 | 2.2 红色血液 |
| 自查要点 | ① 排查战斗/受伤场景是否有写实红色血液 ② 卡通或非写实风格的血液表现可豁免 ③ 若有写实血液，需做和谐处理（如改色、去除） |

### 案例 2.3：游戏含政治敏感内容

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内或商店页含有国家领导人形象、政治敏感议题、历史争议事件、恐怖组织相关内容 |
| 对应违规 | 2.3 涉政内容 |
| 自查要点 | ① 排查游戏角色/NPC 是否涉及真实政治人物 ② 排查剧情是否涉及敏感历史事件 ③ 排查地图/国旗等元素是否有争议 |

### 案例 2.4：游戏内嵌入低俗广告

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内展示的第三方广告含有低俗、色情擦边、欺骗性或强诱导性内容 |
| 对应违规 | 2.4 低俗广告 |
| 自查要点 | ① 检查接入的广告 SDK 展示的广告内容 ② 确认广告平台是否有内容过滤机制 ③ 实际运行游戏查看展示的广告素材 |

## 三、知识产权自查

### 案例 3.1：使用他人美术素材

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏中的角色立绘、场景设计、UI 界面等直接复制或高度相似于他人作品 |
| 对应违规 | 3.1 素材侵权抄袭 |
| 自查要点 | ① 逐一确认所有美术素材的来源和授权 ② 网络下载的免费素材需确认其授权协议允许商用 ③ AI 生成的素材也需确认不侵犯已有作品 |

### 案例 3.2：游戏名或图标模仿知名 IP

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏名称、图标、角色命名等刻意模仿知名 IP 的视觉元素或命名方式 |
| 对应违规 | 3.2 擦边碰瓷热门 IP |
| 自查要点 | ① 检查游戏名是否与知名游戏/动漫/影视 IP 相似 ② 检查图标设计是否借鉴了知名 IP 的视觉风格 ③ 检查游戏内角色命名是否与知名 IP 角色近似 |

### 案例 3.3：使用 IP 但缺少授权证明

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏使用了某知名 IP 的角色、世界观、音乐等内容，但无法提供版权方的授权文件 |
| 对应违规 | 3.3 游戏 IP 授权存疑 |
| 自查要点 | ① 确认所有使用的 IP 内容是否有合法授权 ② 准备好授权证明文件以备审核 |

## 四、分发违规自查

### 案例 4.1：游戏内含外部下载链接

| 项目 | 内容 |
|------|------|
| 场景描述 | 在游戏内弹窗、商店页描述或公告中提供非 TapTap 的下载链接、二维码或文字引导 |
| 对应违规 | 4.1 引导外部包下载 |
| 自查要点 | ① 排查游戏内是否有指向外部下载的链接/二维码 ② 排查商店页描述是否有外部下载引导 ③ 排查开发者公告内容 |

### 案例 4.2：引导用户加群获取安装包

| 项目 | 内容 |
|------|------|
| 场景描述 | 引导用户加入 QQ 群/微信群等社交群组，并在群内提供游戏安装包下载 |
| 对应违规 | 4.2 引导 Q 群下载 |
| 自查要点 | ① 注意：引导加群本身不一定违规，**关键在于群内是否提供安装包下载** ② 排查游戏内/商店页是否有加群引导 + 群内分发安装包的组合行为 |

## 五、运营违规自查

### 案例 5.1：设计了好评引导文案或机制

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内通过奖励（礼包码、道具、积分、现金等）诱导用户好评，或通过弹窗引导用户给予正面评价 |
| 对应违规 | 5.1 引导好评 |
| 自查要点 | ① 排查游戏内是否有"好评送钻石/礼包码"类文案 ② 排查是否有弹窗引导用户去评分且暗示给好评 ③ **关键判定**：强制弹窗本身不违规，但利用弹窗进行好评诱导才违规，判断标准在于内容是否构成好评引导 ④ 排查站外（社交平台等）是否有好评返利活动 |

### 案例 5.2：频繁修改游戏名

| 项目 | 内容 |
|------|------|
| 场景描述 | 短时间内多次修改游戏名称 |
| 对应违规 | 5.2 多次更换游戏名 |
| 自查要点 | ① 确认近期是否频繁改名 ② 避免为蹭热点关键词而反复改名 |

### 案例 5.3：长期使用封闭式测试

| 项目 | 内容 |
|------|------|
| 场景描述 | 将封闭式测试作为长期运营手段，频繁开启或续期测试计划，而非正常使用 |
| 对应违规 | 5.3 高频使用封闭式测试计划 |
| 自查要点 | ① 检查封闭式测试是否已开启过多次/过长时间 ② 考虑改用开放式测试 |

### 案例 5.4：发布多款高度相似游戏（重复入库）

| 项目 | 内容 |
|------|------|
| 场景描述 | 同一开发者发布多款玩法、美术或核心内容高度雷同的游戏，或与平台已有游戏高度相似 |
| 对应违规 | 5.4 游戏雷同重复（别名：重复入库） |
| 自查要点 | ① 检查是否有多款游戏使用相同模板/素材 ② 新游戏是否与自己已上架的游戏高度相似 ③ 跨账号上传雷同游戏可能导致账号注销 |

## 六、资质问题自查

### 案例 6.1：资质文件不匹配

| 项目 | 内容 |
|------|------|
| 场景描述 | 提交的营业执照、软著、ICP 备案等资质文件存在伪造、过期或与游戏信息不一致 |
| 对应违规 | 6.1 资质异常 |
| 自查要点 | ① 确认软著名称与游戏名称一致 ② 确认版号信息与游戏对应 ③ 确认所有资质文件在有效期内 |

### 案例 6.2：无合法企业主体

| 项目 | 内容 |
|------|------|
| 场景描述 | 开发者账号无对应的合法注册企业主体，或分发主体信息缺失 |
| 对应违规 | 6.2 没有分发主体 |
| 自查要点 | ① 确认厂商账号已完成认证 ② 确认企业主体信息真实有效 |

### 案例 6.3：境外游戏无版号

| 项目 | 内容 |
|------|------|
| 场景描述 | 境外开发商的游戏在大陆上线运营但未取得版号 |
| 对应违规 | 6.3 境外无版号 |
| 自查要点 | ① 确认是否为境外开发商身份 ② 若在大陆运营需取得版号 |

### 案例 6.4：版号运营主体已注销

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏版号对应的运营单位企业已被吊销或注销 |
| 对应违规 | 6.4 版号运营单位已吊销 |
| 自查要点 | ① 确认版号运营单位的营业执照是否有效 ② 如已吊销，需联系版号运营单位核实并提供有效授权 |

### 案例 6.5：防沉迷系统缺失或不合规

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏未接入实名认证，或未落实未成年人时长/消费限制 |
| 对应违规 | 6.5 防沉迷不合规 |
| 自查要点 | ① 确认是否接入实名注册认证系统 ② 确认未成年人游戏时长限制是否生效 ③ 确认防沉迷弹窗不可跳过 |

## 七、游戏质量自查

### 案例 7.1：游戏崩溃或无法运行

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏存在频繁崩溃、卡死、无法加载、核心功能无法使用 |
| 对应违规 | 7.1 游戏不能正常游玩 |
| 自查要点 | ① 在多个设备上测试游戏运行稳定性 ② 确认核心玩法流程可以正常完成 ③ 特别关注低端设备的兼容性 |

### 案例 7.2：聊天系统未过滤敏感词

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏内聊天、昵称输入等用户生成内容场景，未部署有效的敏感词过滤 |
| 对应违规 | 7.2 未屏蔽敏感词 |
| 自查要点 | ① 确认聊天/昵称系统是否接入敏感词过滤 ② 测试敏感词能否被有效拦截 ③ 过滤系统不能形同虚设 |

### 案例 7.3：批量发布套壳游戏

| 项目 | 内容 |
|------|------|
| 场景描述 | 使用相同模板或引擎简单套壳，批量发布缺乏独立内容的低质量游戏 |
| 对应违规 | 7.3 大量低质游戏 → 8.2 低质厂商处罚 |
| 自查要点 | ① 确认每款游戏有独立的核心内容和玩法 ② 避免仅换皮/改名就发布新游戏 |

### 案例 7.4：整体复制已有游戏

| 项目 | 内容 |
|------|------|
| 场景描述 | 游戏对已有游戏的核心玩法、美术资产、代码等进行整体或大量复制 |
| 对应违规 | 7.4 盗版抄袭 → 8.1 盗版抄袭厂商处罚 |
| 自查要点 | ① 确认游戏核心玩法有原创性 ② 确认美术资产非直接复制他人作品 ③ 确认代码非反编译/逆向获取 |

## 快速自查清单

提审前按此清单逐项确认：

| 序号 | 检查项 | 对应违规类型 |
|------|--------|-------------|

| 2 | 游戏内/商店页是否有色情、性暗示内容？ | 2.1 色情内容 |
| 3 | 游戏内是否有写实红色血液/血腥画面？ | 2.2 红色血液 |
| 4 | 游戏内/商店页是否有政治敏感内容？ | 2.3 涉政内容 |
| 5 | 游戏内广告是否有低俗/欺骗性内容？ | 2.4 低俗广告 |
| 6 | 所有美术素材是否有合法授权？ | 3.1 素材侵权 |
| 7 | 游戏名/图标是否模仿知名 IP？ | 3.2 碰瓷 IP |
| 8 | 使用的 IP 内容是否有授权证明？ | 3.3 IP 授权存疑 |
| 9 | 是否有外部下载链接/二维码？ | 4.1 外部包下载 |
| 10 | 是否引导加群且群内分发安装包？ | 4.2 Q 群下载 |
| 11 | 是否有好评返利/好评诱导文案？ | 5.1 引导好评 |
| 12 | 近期是否频繁修改游戏名？ | 5.2 频繁改名 |
| 13 | 封闭式测试是否使用过于频繁？ | 5.3 滥用测试 |
| 14 | 是否发布了多款高度相似的游戏？ | 5.4 雷同重复 |
| 15 | 资质文件（软著/版号/执照）是否真实有效且匹配？ | 6.1 资质异常 |
| 16 | 是否有合法的企业分发主体？ | 6.2 无分发主体 |
| 17 | 境外游戏是否取得版号？ | 6.3 境外无版号 |
| 18 | 版号运营单位是否仍在营业？ | 6.4 运营单位吊销 |
| 19 | 防沉迷系统是否完整且不可跳过？ | 6.5 防沉迷不合规 |
| 20 | 游戏是否能正常运行？ | 7.1 无法游玩 |
| 21 | 聊天/昵称系统是否有敏感词过滤？ | 7.2 未屏蔽敏感词 |

| 23 | 游戏是否原创（非整体复制）？ | 7.4 盗版抄袭 |
