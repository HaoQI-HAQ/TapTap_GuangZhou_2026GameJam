# violation-notify — TapTap 游戏合规助手

帮助 TapTap 开发者理解和应对游戏违规问题，提供违规识别、整改指引、申诉指导、提审前自查，以及 build 前自动代码扫描。

---

## 功能概览

本 Skill 包含三大功能模块，覆盖违规处理的完整生命周期：

| 模块 | 触发方式 | 说明 |
|------|---------|------|
| 违规响应 | 对话中描述违规情况 | 识别违规类型、告知后果、提供整改与申诉指引 |
| 提审自查 | 对话中要求自查/合规检查 | 基于 23 项自查清单逐项排查，输出风险报告 |
| 自动扫描 | 每次 build 前自动执行 | 扫描代码中的违规关键词，输出扫描报告 |

---

## 功能详情

### 1. 违规响应

当开发者收到违规通知或想了解某行为是否违规时，自动触发。

**处理流程：**

1. **识别违规类型** — 从 8 大类 26 种子类型中精准匹配
2. **说明后果** — 引用官方文档，告知处罚措施和整改要求
3. **整改指引** — 基于官方整改要求，给出可操作的处理建议
4. **申诉指导** — 如开发者认为误判，指导准备材料并通过开发者中心提交申诉
5. **预防建议** — 帮助避免再次触发违规

**触发关键词：** 违规、下架、处罚、整改、申诉、封禁、合规、流量受限

**覆盖的违规类型：**

| 大类 | 子类型 |
|------|--------|
| 计费违规 | 偷开计费、多次偷开计费 |
| 内容违规 | 色情内容、红色血液、涉政内容、低俗广告 |
| 知识产权 | 素材侵权抄袭、擦边碰瓷热门 IP、游戏 IP 授权存疑 |
| 分发违规 | 引导外部包下载、引导 Q 群下载 |
| 运营违规 | 引导好评、多次更换游戏名、高频使用封闭式测试计划、游戏雷同重复 |
| 资质问题 | 资质异常、没有分发主体、境外无版号、版号运营单位已吊销、防沉迷不合规 |
| 游戏质量 | 游戏不能正常游玩、未屏蔽敏感词、大量低质游戏、盗版抄袭 |
| 账号/厂商级 | 盗版抄袭厂商处罚、低质厂商处罚 |

此外还涵盖**流量受限**（非违规处罚，属于 Maker 游戏默认的分发限制状态）的说明与解除指引。

---

### 2. 提审自查

当开发者准备提审、想主动排查违规风险时触发。

**处理流程：**

1. 加载自查案例库（23 项经典违规场景）
2. 确认自查范围 — 可全面检查或只查特定类别
3. 逐项引导排查 — 展示场景描述和自查要点，给出判断
4. 汇总风险项 — 列出问题、对应违规类型、可能后果
5. 输出整改建议

**触发关键词：** 自查、合规检查、提审前检查、排查违规

**自查清单覆盖 8 大类别：**

- 计费合规（3 项）
- 内容合规（4 项）
- 知识产权（3 项）
- 分发合规（3 项）
- 运营合规（3 项）
- 资质合规（3 项）
- 游戏质量（3 项）
- 商店页素材（1 项）

可按需使用完整清单，也可针对特定类别单独检查。

---

### 3. 自动扫描

通过 Claude Code Hook 机制，在每次调用 `build` 工具前自动执行代码扫描。

**检测项目：**

| 检测类别 | 对应违规类型 | 检测方式 |
|---------|------------|--------|
| 计费关键词 | 1.1 偷开计费 | 匹配充值/内购/付费/支付等关键词 |
| 外部下载引导 | 4.1 引导外部包下载 | 匹配 .apk/.ipa/下载链接等关键词 |
| 社交群组引导 | 4.2 引导 Q 群下载 | 匹配 QQ群/群号/加群等关键词 |
| 好评引导 | 5.1 引导好评 | 匹配好评/五星/评分送等关键词 |
| 敏感词过滤缺失 | 7.2 未屏蔽敏感词 | 检测有聊天功能但无敏感词过滤 |
| 外部链接 | 综合检查 | 检测非平台域名的外部 URL |

**扫描特性：**

- 无风险时输出 `[OK]`，build 正常继续
- 有风险时输出详细报告（文件名、行号、匹配内容），build 仍然继续（不阻止构建）
- 支持 `-- violation-scan-ignore` 行内注释抑制误报

## 附带工具

### 合规自查报告模板

提供标准化的提审前自查报告模板，包含：

1. 项目基本信息
2. 自动扫描结果记录
3. 七大类逐项检查表（可打勾）
4. 商店页素材完整性检查
5. 风险汇总表
6. 最终确认清单

开发者可让 AI 按此模板自动生成报告，用于团队内部存档或提审前确认。

## 核心设计原则

1. **官方文档为唯一信息源** — 所有违规定义、处罚标准、整改要求均引用自 TapTap 官方文档，禁止编造或推测
2. **不给规避建议** — 作为平台官方助手，不引导开发者绕开规则边界
3. **信息不足时如实告知** — 文档未覆盖的问题，引导开发者联系平台获取官方答复
4. **提醒但不阻止** — 自动扫描发现风险时提供报告，但不阻止构建流程

## 文件结构

```
violation-notify/
├── SKILL.md                        # Skill 主定义（触发条件、执行步骤、规则）
├── README.md                       # 本文件
├── scan.sh                         # build 前自动扫描脚本
└── references/
    ├── violation-templates.md      # 违规模板库（8大类26种子类型，结构化整理）
    ├── self-check-cases.md         # 自查案例库（23项检查清单）
    └── compliance-report-template.md  # 合规自查报告模板
```

Hook 配置（需放在项目根目录）：

```
.claude/settings.json               # PreToolUse hook，build 前触发 scan.sh
```

## 使用方式

部署到项目后，无需任何特殊命令，在对话中自然使用即可：

| 场景 | 示例 |
|------|------|
| 收到违规通知 | "我的游戏收到偷开计费的通知，怎么处理？" |
| 确认行为是否违规 | "游戏里引导玩家加 QQ 群算违规吗？" |
| 提审前自查 | "帮我做一下提审前合规检查" |
| 了解流量受限 | "我的游戏流量受限了怎么解除？" |
| 自动扫描 | 正常 build，扫描自动执行 |

---

## 局限性

- **代码扫描基于关键词匹配**，存在误报可能，需结合实际情况判断
- **无法检测画面内容**（色情/血腥/涉政需人工确认）
- **无法验证素材版权**（需开发者自行确认授权）
- **无法核查资质状态**（需在开发者中心确认）
- **政策可能更新**，建议定期关注平台最新协议和公告
